# 📚 TestGo Skills 使用指南

本项目为 NAS 产品测试工程师提供了一套完整的测试用例生成和管理工具。

## 🚀 可用快捷命令（Skills）

### 1️⃣ 测试用例生成系列

#### `/功能测试用例`
将需求文档转换为功能测试用例（xlsx 格式）

**用法：**
```
/功能测试用例 需求文档.md
/功能测试用例 <直接粘贴需求文本>
```

**覆盖范围：**
- P0：核心正向场景
- P1：有效/无效等价类、边界值、特殊字符、并发、权限等
- P2：数据异常、设备重启

---

#### `/兼容性测试用例`
将需求文档转换为兼容性测试用例（xlsx 格式）

**用法：**
```
/兼容性测试用例 需求文档.md
```

**覆盖范围：**
- P1：平台、浏览器、操作系统、终端、网络环境
- P2：弱网、断网处理

---

#### `/专项测试用例`
将需求文档转换为专项测试用例（xlsx 格式）

**用法：**
```
/专项测试用例 需求文档.md
```

**覆盖范围：**
- P2：存储池异常（拔盘、降级、损毁）
- P2：安全测试（XSS、SQL注入、鉴权等）
- P2：性能与压力测试

---

#### `/升级刷机测试用例`
将需求文档转换为升级刷机测试用例（xlsx 格式）

**用法：**
```
/升级刷机测试用例 需求文档.md
```

**覆盖范围：**
- P1：数据库脚本变更
- P1：OTA 升级验证
- P1：新版本刷机初始化

---

### 2️⃣ Bugfix验证

#### `/Bugfix验证用例`
将Bug修复信息转换为高效可靠的验证用例（xlsx 格式）

**用法：**
```
/Bugfix验证用例 Bug修复报告.md
/Bugfix验证用例 <直接描述Bug和修复方案>
```

**验证模型（三层）：**
- P0 根因验证：用DevTools/日志/数据库等客观手段验证根本原因已消除
- P1 症状验证：复现原始Bug路径，确认症状消失
- P1/P2 回归验证：验证修复未影响相关功能

**效率控制：** 总用例数 4-11 条，每条预期结果必须可量化、可工具化、可对比

**验证模式库：** 内置可复用的验证模式（Pattern），遇到同类Bug自动匹配

---

### 3️⃣ 测试用例评审

#### `/测试用例评审`
评审现有测试用例的完整性和规范性

**用法：**
```
/测试用例评审 测试用例.xlsx
/测试用例评审 测试用例.md
```

**评审维度：**
- 规范性：字段完整性、格式正确性
- 完整性：步骤清晰度、结果对应性
- 可执行性：是否可独立执行
- 覆盖度：需求覆盖、场景覆盖

---

### 4️⃣ 项目管理工具

#### `/structure`
快速查看当前项目的目录结构

**用法：**
```
/structure
```

**功能：**
- 以树形结构展示项目目录
- 自动添加文件说明
- 显示统计信息

---

#### `/更新问答记录`
将当前对话的问答内容追加到 conversation-qa.md

**用法：**
```
/更新问答记录
```

**功能：**
- 自动整理当前对话问答
- 问题用英文记录，答案用中文记录
- 保持格式一致性

## 💡 使用技巧

### 批量生成测试用例

如果需要为同一需求文档生成多种类型的测试用例，可以连续调用：

```
/功能测试用例 需求.md
/兼容性测试用例 需求.md
/专项测试用例 需求.md
```

会生成三个独立的 xlsx 文件：
- `需求_功能测试用例.xlsx`
- `需求_兼容性测试用例.xlsx`
- `需求_专项测试用例.xlsx`

### Bugfix快速验证

修复Bug后，生成验证用例并评审：

```
/Bugfix验证用例 修复报告.md
/测试用例评审 项目_Bug描述_Bugfix验证用例.xlsx
```

### 快速评审

生成用例后，立即调用评审命令检查质量：

```
/功能测试用例 需求.md
/测试用例评审 需求_功能测试用例.xlsx
```

### 查看项目结构

快速了解项目文件组织：

```
/structure
```

### 记录对话

在完成一轮重要的对话后，记得保存记录：

```
/更新问答记录
```

## 🔧 自定义配置

### 修改权限设置

编辑 `.claude/settings.local.json` 可以调整权限策略。

### 添加新的 Skill

在 `.claude/skills/` 目录下创建新的 `.md` 文件，按照以下格式编写：

```markdown
---
name: Skill 名称
description: Skill 描述。用法：/Skill名称 <参数>
---

Skill 的详细 Prompt 内容
```
